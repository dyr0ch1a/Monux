# Monux — Project Context

## Что это за проект
Monux — workspace на Rust с тремя крейтами:
- `core`: доменная логика заметок, индекс, файловое хранилище, редакторные абстракции.
- `cli`: интерфейс командной строки (`init/new/list/find/delete/rename/sync/edit/tags`).
- `tui`: терминальный UI на `ratatui` + `crossterm`.

Проект решает задачу локального управления заметками в markdown-формате с быстрым доступом через CLI и TUI.

## Текущее состояние (по структуре и коду)
- Монорепозиторий на Cargo workspace, модульная раскладка аккуратная.
- `core/index.rs` использует `redb` для индекса заметок (slug, title, tags, id).
- Поддерживаются операции создания, поиска, тегов, переименования и удаления.
- CLI покрывает базовый сценарий работы со знаниями; README даёт quick start.
- TUI имеет базовый event-loop и отрисовку, но выглядит как ранняя стадия.

## Что лучше улучшить в первую очередь

### 1) Надёжность данных и консистентность
- Добавить проверки консистентности между индексом и файловой системой в `sync`.
- Сделать recovery-команду (например, `monux doctor` / `monux repair`) для восстановления битых ссылок в индексе.
- Явно оформить стратегию конфликтов slug/rename (детерминированное поведение + понятные ошибки).

### 2) Тесты
- Добавить unit-тесты для `normalize_note_key`, обработки тегов, rename/delete сценариев.
- Добавить интеграционные тесты CLI-команд через временный каталог (`tempdir`) и реальные markdown-файлы.
- Проверять edge-cases: пустые имена, unicode, вложенные пути, дубликаты тегов.

### 3) UX CLI
- Единый формат вывода (человекочитаемый и `--json` для автоматизации).
- Улучшить сообщения об ошибках: что пошло не так, почему, и как исправить.
- Добавить команды качества жизни: `show`, `open`, `export`, фильтрацию `list` по тегам/дате.

### 4) TUI как полноценный клиент
- Разделить state management и rendering по более явным слоям.
- Добавить keymap/help-overlay и статусную строку действий.
- Реализовать быстрый поиск, просмотр заметки и inline-операции с тегами.

### 5) Инженерная гигиена
- CI pipeline: `fmt`, `clippy`, тесты на каждый PR/коммит.
- Уточнить документацию: архитектура, жизненный цикл заметки, структура данных индекса.
- Закрепить версионирование и changelog (даже минимальный).

## Предлагаемый порядок работ
1. Покрыть `core` тестами и закрепить ожидаемое поведение.
2. Усилить `sync` и добавить проверку/repair консистентности.
3. Выровнять UX CLI (`--json`, ошибки, стабильный output).
4. После стабилизации логики расширять TUI.

## Критерий «готово» для следующей версии
- Базовые сценарии заметок проходят интеграционные тесты.
- Индекс и файловая система не расходятся после sync/rename/delete.
- CLI выдаёт стабильный предсказуемый вывод.
- TUI поддерживает полный read/write workflow без необходимости постоянно выходить в CLI.
